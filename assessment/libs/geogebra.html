addGeogebra

addGeogebra(applet,[width,height,commands,params,callbacks,qn,part])

place a geogebra HTML5 applet.

applet should be one of:

  • "graphing", "geometry", "3d", "spreadsheet", "classic", or "probability" for a blank applet in one of those modes
  • The applet ID for an app uploaded to geogebra.org. For example, if the URL for the applet is https://www.geogebra.org/m/ws7xbhyh, the applet ID is "ws7xbhyh". You can use applets others have uploaded, or ones you upload. To give proper attribution, include in the question "Other Attribution" something like "Geogebra applet created by ____" with the creator's name.
  • The base64ggb for an applet you created. In the applet editor, you grab this by pressing Ctrl+Shift+B

width and height are the dimensions in pixels for the applet. Default is 400 by 200.

commands is an optional array of javascript commands to execute on the geogebra applet to change the contents. See GeoGebra Apps API for available commands. For example, ["evalCommand('A = ($a,$b)')","evalCommand('B = ($c,$d)')"] would set the value of points A and B to the provided coordinates.

params is an optional key=>value array of parameters, like ['framePossible'=>'true','showToolBar'=>'true']. See GeoGebra App Parameters for options

callbacks is an optional array that allows you to pull values from Geogebra on submit. The values will be stored as a comma-separated list in the answerbox corresponding with qn and part. Usually you will want to use $thisq for qn. part is not required for single-part questions. For example, ["getValue('a')","getValue('b')"] would get the values of Geogebra variables a and b and store them as a comma separated list
